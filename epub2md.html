EPUB to Markdown

Convert an EPUB to Markdown in reading order.

About & limitations

What this does: Reads the EPUB's container.xml to find the OPF package, walks the spine in reading order, and emits Markdown.

Preserved: headings, paragraphs, bold, italic, strikethrough (~~x~~), superscript / subscript (^x^ / ~x~), inline code, fenced code blocks, lists (nested), blockquotes, links, tables, horizontal rules, and footnotes ([^id]) when the source uses EPUB epub:type="noteref" / epub:type="footnote" attributes.

TOC links: cross-chapter links like <a href="chapter01.html#sec2"> are rewritten to same-document heading anchors. The renderer resolves fragments to the slug of the nearest enclosing heading (or the chapter's first heading if nothing matches). When a headingless chapter's title is encoded as an image or styled prose and its text matches a link pointing at that chapter, that element is promoted to a level-2 heading so the TOC resolves.

Images: emitted as ![alt](images/filename). If the EPUB contains referenced images, the output is a .zip containing the .md alongside an images/ folder so the links resolve. Books with no images download as a single .md. Basenames are deduplicated across chapters.

Stripped: ruby / furigana (base text kept, reading dropped), underlines, font-size changes, SVG, MathML, arbitrary CSS styling, and any HTML with no Markdown equivalent.

DRM: DRM-protected EPUBs (Adobe ADEPT, Apple FairPlay, Kindle KFX) cannot be read.

Privacy: Everything runs in your browser. Your file never leaves your device.

All data stays on your device.
